Estructura Maestra de Prompt

1. Persona
En lugar de decir "ayúdame", asígnale un rol de alto nivel.

Ejemplo: "Actúa como un Arquitecto de Software Senior y Líder de Producto con 15 años de experiencia en desarrollo ágil y escalabilidad."

2. Tarea
Define la acción principal de forma directa.

Ejemplo: "Diseña la arquitectura técnica y el roadmap de desarrollo para una aplicación web de gestión de inventarios para pequeñas empresas."

3. Contexto
Aquí es donde eliminas la vaguedad. Dale los "porqués" y las restricciones.

Ejemplo: "El software debe ser fácil de usar para personas sin conocimientos técnicos. Necesito que use un stack moderno (como React y Node.js). El objetivo es que soporte hasta 1,000 usuarios concurrentes y se integre con APIs de pago locales en Chile."

4. Formato
Dile exactamente cómo quieres recibir la información.

Ejemplo: "Entrega la respuesta organizada en:

Lista de funcionalidades (MVP).

Diagrama lógico (en formato de texto o Markdown).

Sugerencia de Base de Datos con justificación.

Primeros 3 pasos técnicos para empezar el repositorio."


Esto sería el texto completo. 

Actúa como un Arquitecto de Software Full Stack. Tu tarea es crear el diseño técnico de una plataforma de automatización de flujos de trabajo con IA (estilo n8n). Como contexto, el sistema debe permitir a usuarios no técnicos conectar su Gmail con modelos de lenguaje para resumir correos automáticamente; prioriza la seguridad de los datos y el bajo costo de servidor. El formato de salida debe ser una tabla con los módulos necesarios, seguida de un ejemplo de código en Python para la conexión principal.
